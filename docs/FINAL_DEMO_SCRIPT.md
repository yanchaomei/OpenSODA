## 🎤 详细脚本

### 第一部分：开场 (0:00-0:30)

**【旁白】**
> 各位评委老师好！

> 我们的作品是 **OpenSource Copilot** —— 一个基于 ReAct Agent 的开源社区智能运营助手。

> 1. **ReAct 架构** —— Agent 能推理、行动、观察，像人类专家一样工作

### 第三部分：功能演示 (1:00-3:30) ⭐ 重点

#### 3.1 智能对话分析 (1:00-2:00)

**【操作】** 在首页点击"开始对话"，输入：

**【旁白】**
> 现在我用自然语言提问。注意看右侧的 Agent 思考过程：
> 
> - 🤔 首先理解问题意图
> - 🔧 决定调用 analyze_repo_health 工具
> - 📊 从 OpenDigger 获取数据
> - 💬 生成专业的分析报告
> 
> 这就是 **ReAct 模式** —— 推理与行动交替进行。

**【画面】** 等待结果出现

**【旁白】**
> 分析完成！可以看到：
> - **综合健康度评分** 72 分
> - **四个维度**的雷达图
> - **AI 生成的一句话总结**
> - 具体的**亮点和改进建议**

### 第四部分：技术亮点 (3:30-4:30)

**【画面】** 切换到 PPT 架构图页面

**【旁白】**
> 下面介绍几个技术亮点：

#### 4.1 LangGraph 状态机

> 我们用 LangGraph 构建了完整的 ReAct 循环：
> - START → Agent 推理 → 判断是否需要工具
> - 需要 → 执行工具 → 回到 Agent
> - 不需要 → 输出结果 → END

#### 4.2 流式输出

> Agent 的思考过程是**实时流式输出**的，用户能看到 AI 在"想什么"、"做什么"。

#### 4.3 智能缓存

> 为了提升性能，我们实现了两级缓存：
> - L1 内存缓存
> - L2 Redis 缓存
> 
> 相同查询从 3 秒降到 50 毫秒。

## 🔧 备用方案

### 网络故障

如果在线演示网络不稳定：

1. **切换到本地演示**
   ```bash
   cd OpenSODA
   docker-compose up -d
   # 访问 http://localhost:3000
   ```

2. **使用预录视频**
   - 准备一段 3 分钟的演示视频作为备份

### API 调用失败

如果 OpenAI API 超时：

1. 展示**预缓存的分析结果**
2. 说明："我们预置了热门项目的分析结果，确保演示流畅"

### 演示数据推荐

建议使用以下项目演示（数据稳定）：

| 项目 | 特点 |
|------|------|
| `apache/dubbo` | 分数中等，有改进空间 |
| `vuejs/vue` | 分数较高，亮点多 |
| `X-lab2017/open-digger` | 赛题相关，有意义 |
